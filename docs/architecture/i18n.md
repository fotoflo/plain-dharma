# Internationalization (i18n) — Plain Dharma

*Last updated: 2026-05-28*

Multi-locale support for Plain Dharma: all six suttas translated to Chinese (Mandarin), with locale-aware routing, UI strings, and narration. English remains the default; new locales can be added without code changes once the content registry is extended.

## Supported locales

```ts
export const SUPPORTED_LOCALES = ["en", "zh"] as const;
export const DEFAULT_LOCALE: Locale = "en";
```

English is the default and serves from the root (`/`). Chinese routes live under `/zh/...`. Additional locales (Thai, Vietnamese, etc.) can be added by extending `SUPPORTED_LOCALES` and populating the registry described below.

## Routing

**English** (default): routes at the root
- `/` — home
- `/first-talk`, `/not-self`, etc. — per-sutta pages
- `/read` — combined reading
- `/about`, `/glossary`, `/download` — static pages

**Chinese**: routes under `/zh/`
- `/zh` — Chinese home
- `/zh/first-talk`, `/zh/not-self`, etc. — Chinese per-sutta pages
- `/zh/read` — combined reading in Chinese
- `/zh/about`, `/zh/glossary`, `/zh/download` — static pages

### Route structure (not `[locale]`, literal `/zh/`)

The app tree uses a literal `/zh` directory under `src/app/`, not a dynamic `[locale]` segment. This avoids conflicts with the sutta slug routing at the root level.

```
src/app/
  ├── [slug]/page.tsx         # EN sutta pages (first-talk, not-self, ...)
  ├── read/page.tsx           # EN /read
  ├── about/page.tsx          # EN /about
  ├── zh/
  │   ├── [slug]/page.tsx     # ZH sutta pages (/zh/first-talk, etc.)
  │   ├── read/page.tsx       # ZH /read
  │   ├── about/page.tsx      # ZH /about
  │   └── ...
  └── ... (other routes)
```

Each route page imports the appropriate view component and passes the `locale` prop explicitly.

## Content registry

### `src/content/index.ts`

Three-part registry: base metadata (Pali names, ordinal), locale-specific display strings (titles, subtitles, teasers), and dynamic loaders.

**`SUTTA_BASE`** — invariant metadata per sutta:
```ts
{
  slug: SuttaSlug;
  ordinal: number;
  pali_name: string;    // Pali original, not translated
}
```

**`SUTTA_DISPLAY`** — locale-specific strings:
```ts
Record<Locale, Record<SuttaSlug, {
  title: string;
  subtitle: string;
  teaser: string;
  kicker_override?: string;  // Optional: canonical name in that locale
}>>
```

The `kicker_override` field allows a locale to display its own canonical title (e.g., Chinese `转法轮经` Zhuǎn Fǎ Lún Jīng "Turning the Wheel of Dharma") in place of the Pali name. Falls back to `pali_name` when unset.

**`LOADERS`** — dynamic MDX imports:
```ts
Record<Locale, Record<SuttaSlug, () => Promise<MDXContent>>>
```

Must be exhaustive: TypeScript errors if any sutta × locale pair is missing. Adding a new locale means adding a complete inner record with all six suttas.

**Helpers:**
- `getMeta(locale, slug)` — merges `SUTTA_BASE` + `SUTTA_DISPLAY[locale][slug]`
- `getSuttasInOrder(locale)` — returns all six suttas in canonical order with merged metadata
- `getAvailableLocales(slug)` — returns locales that have a loader registered for that slug

### `src/content/strings.ts`

UI strings and metadata for all locales:
```ts
export function getStrings(locale: Locale): {
  nav: { ... }
  footer: { ... }
  audio: { listen, pause, play, prev, next, ... }
  home: { tagline, cta, ... }
  read: { ... }
  about: { ... }
  glossary: { title, subtitle, ... }
  canonicalLinks: { label, access, ... }
  // ... more
}
```

All client-facing labels, aria-labels, template strings (with `{title}`, `{n}`, `{time}` placeholders), and metadata descriptions come from here. Components fetch via `getStrings(locale)` and never hardcode strings.

### `src/content/drops.ts`

Editorial one-liners and `/read` framing prose, locale-aware:
```ts
DROPS[locale][slug]          // one-liner after each article
PREFACE[locale]              // intro text at top of /read
CLOSING[locale]              // outro text at bottom of /read
```

### `src/content/canonical-links.ts`

Pali references and translation links, differing by locale:
```ts
CANONICAL_LINKS[locale][slug] = {
  label: string;
  links: { title: string; url: string }[]
}
```

EN links point to Access to Insight + SuttaCentral; ZH links point to SuttaCentral Āgama and CBETA Taishō.

## Locale-aware routing helpers

`src/lib/locale-href.ts` provides URL building and locale detection:

```ts
localizedHref(locale, path)           // "/zh/first-talk" if locale="zh"; "/first-talk" if locale="en"
getLocaleFromPathname(pathname)       // "/zh/first-talk" → "zh"; "/first-talk" → "en"
stripLocalePrefix(pathname)           // "/zh/first-talk" → "/first-talk" (logical path)
switchLocalePath(pathname, locale)    // "/zh/first-talk" + "en" → "/first-talk"; vice versa
```

Used by the locale switcher and view components to construct language-aware links.

## View components

Extracted view components in `src/views/` accept a `locale` prop and compose the page:

- `HomeView` — home page listing all six suttas
- `SuttaView` — single sutta page with title, content, audio player, footnotes
- `ReadView` — combined all-six reading with cross-sutta audio playback
- `AboutView` — About page
- `GlossaryView` — Glossary with locale-specific definitions

Each view calls `getStrings(locale)` for UI labels and `getMeta(locale, slug)` for content metadata.

Routes pass locale explicitly:
```tsx
// src/app/[slug]/page.tsx (EN)
export default function Page({ params }) {
  const locale = "en";
  return <SuttaView locale={locale} slug={params.slug} />;
}

// src/app/zh/[slug]/page.tsx (ZH)
export default function Page({ params }) {
  const locale = "zh";
  return <SuttaView locale={locale} slug={params.slug} />;
}
```

## Header, Footer, and component locale awareness

Components that render locale-aware UI (navigation links, locale switcher, footer copyright notices) use `usePathname()` from Next.js to derive the current locale via `getLocaleFromPathname()`:

```tsx
export function Header() {
  const pathname = usePathname();
  const locale = getLocaleFromPathname(pathname);
  const s = getStrings(locale);

  return (
    <header>
      <nav>
        <Link href={localizedHref(locale, "")}>Home</Link>
        <Link href={localizedHref(locale, "read")}>Read</Link>
        <LocaleSwitcher locale={locale} pathname={pathname} />
      </nav>
    </header>
  );
}
```

`LocaleSwitcher` renders a language toggle (EN ↔ ZH) and uses `switchLocalePath()` to transition between locale versions of the current page.

### Partial manifest support (in-progress audio)

`src/content/audio.ts` loads per-sutta manifests and stitches them into a combined playlist for `/read`. If a sutta's manifest is missing (e.g., audio still being recorded), it is **skipped** rather than causing a build failure. The combined manifest returns all available sections in order.

```ts
// getCombinedAudioManifest(locale)
const perSutta = await Promise.all(SUTTAS.map(slug => getAudioManifest(locale, slug)));
const available = perSutta.filter(m => m !== null);  // Skip missing
// Stitch available manifests into one playlist
```

This allows recording audio for one sutta while keeping others playable — no blocking on the final sutta.

## Audio voice and tempo decisions

**English narration:**
- **Voice:** Sage — a contemplative, steady male voice
- **Model:** OpenAI `gpt-4o-mini-tts`
- **Tempo:** ffmpeg `atempo=0.8333` post-process applies a 20% meditative slowdown (time stretching without pitch shift)

**Chinese (Mandarin) narration:**
- **Voice:** Carter (`bU2VfAdiOb2Gv2eZWlFq`) — a meditative, measured male voice
- **Model:** ElevenLabs `eleven_multilingual_v2` (multilingual support for mixed Pali/CJK text)
- **Prosody tuning:** `stability=0.75, style=0.1, similarity=0.75` (lower stability + minimal style = steady, expressive base)
- **Tempo:** ffmpeg `atempo=0.7692` post-process applies a 30% meditative slowdown (time stretching without pitch shift)

Both locale-specific narrations are time-stretched to create a calm, unhurried listening experience suited to contemplative reading of the sutras.

## Adding a new locale

1. **Register the locale:**
   - Add to `SUPPORTED_LOCALES` in `src/content/index.ts`

2. **Add content:**
   - Create `src/content/{locale}/first-talk.mdx`, `not-self.mdx`, etc. (all six)
   - Add translations to `SUTTA_DISPLAY[locale]` in `index.ts`
   - Add `LOADERS[locale]` with a complete record of all six slugs
   - Extend `strings.ts` with translations of all UI strings

3. **Create routes:**
   - Create `src/app/{locale}/[slug]/page.tsx`, `src/app/{locale}/read/page.tsx`, etc.
   - Pass the locale to view components

4. **Audio (optional):**
   - Run `pnpm generate-audio [slug] [locale]` to create audio files
   - Audio narration is optional; pages render without it if manifests are missing

5. **Canonical links and glossary (optional):**
   - Extend `canonical-links.ts` with sources specific to that locale
   - Update glossary terms in `strings.ts`

See `.claude/skills/translate-sutta/SKILL.md` for a full step-by-step pipeline covering content translation, audio generation, and integration.

## Locale switching

Users can switch locales via the `LocaleSwitcher` component in the header. Switching from `/zh/first-talk` to English goes to `/first-talk` (same logical page, different locale).

Internally: `switchLocalePath(pathname, targetLocale)` → `stripLocalePrefix()` → `localizedHref(targetLocale, logical)`.

## Important patterns and gotchas

**`LOADERS` is exhaustive.** TypeScript will error if any locale is missing a slug. This is intentional — the build should fail fast if a translation is incomplete.

**`getStrings(locale)` is server-only** — it cannot be imported into Client Components. Components that need strings must accept a `locale` prop and call `getStrings` at the route level (RSC), then pass strings or string-fetching functions down.

**`getMeta()` and `getSuttasInOrder()` require locale.** Always pass the locale explicitly.

**Pali names never translate.** `pali_name` lives in `SUTTA_BASE` and does not change per locale. The `kicker_override` field in `SUTTA_DISPLAY` allows a locale to _replace_ the Pali name in the UI, but the Pali original is canonical.

**`stripLocalePrefix()` handles both `/zh/...` and `/zh`.** The root `/zh` path strips to `/`, preserving the "home page" semantics.

**Canonical links differ by locale.** The mapping lives in `canonical-links.ts` and is read by `getStrings()` so it's available to UI components. English points to Access to Insight; Chinese points to SuttaCentral Āgama and CBETA Taishō.

**Partial manifests are safe.** If audio for one sutta is still recording, `getCombinedAudioManifest()` skips it and returns a playlist with the other five. The player renders the available sections. This unblocks partial shipping of audio content across locales.
