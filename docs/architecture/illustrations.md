# Illustrations — Plain Dharma

*Last updated: 2026-05-26*

One illustration per teaching. Generated by Gemini's image model as gestural
ink-line drawings on a saffron watercolor wash, then post-processed to
transparent backgrounds so they composite cleanly over the cream or navy page.

## Overview

```
scripts/generate-illustrations.ts
  → Gemini REST API (gemini-2.5-flash-image model family)
  → public/illustrations/{slug}.png  (raw, white background)

scripts/transparentize-illustrations.ts
  → ImageMagick (preferred) or sharp (fallback)
  → public/illustrations/{slug}.png  (in-place, transparent background)
  → public/illustrations/{slug}-dark.png  (dark-mode variant, if present)

src/content/illustrations.ts
  → getIllustrationUrl(slug)      → /illustrations/{slug}.png?v=<mtime>
  → getIllustrationDarkUrl(slug)  → /illustrations/{slug}-dark.png?v=<mtime>

src/components/SuttaIllustration.tsx
  → renders <Image> light + dark pair, toggled by CSS .dark class
```

## Key files

| File | Role |
|---|---|
| `scripts/generate-illustrations.ts` | Calls Gemini REST API; skips existing files |
| `scripts/transparentize-illustrations.ts` | Alpha-fades near-white pixels; writes in-place |
| `src/content/illustrations.ts` | `getIllustrationUrl` / `getIllustrationDarkUrl` — mtime-versioned URLs |
| `src/components/SuttaIllustration.tsx` | Next.js `<Image>` wrapper with light/dark pair |
| `public/illustrations/` | Output directory (six PNGs + optional `-dark` variants) |
| `.env.local` | `GOOGLE_GENERATIVE_AI_KEY` required by the generation script |
| `next.config.ts` | `images.localPatterns` allowlist for `?v=` query strings |

## Generation

```
pnpm generate-illustrations
```

Reads `GOOGLE_GENERATIVE_AI_KEY` from `.env.local` (passed via
`node --env-file=.env.local`). Skips any slug whose `.png` already exists —
re-runs are safe and don't burn API credits.

The script tries a list of model candidates in order (`gemini-2.5-flash-image`,
`gemini-2.5-flash-image-preview`, `gemini-2.0-flash-preview-image-generation`)
because Google's image model names have changed frequently. Auth errors abort
the model loop immediately.

Style prompt: single gestural ink line (Matisse / Saul Steinberg aesthetic) +
prominent saffron watercolor wash (#C7651C) + cream background (#F5EFE0). No
Buddhist iconography — universal/humanist imagery only.

## Transparency pipeline

```
pnpm transparentize-illustrations
```

Gemini returns a near-white background (~#FCFCFC) that doesn't match the page
paper (#F5EFE0). This script makes it transparent:

- **Detection:** pixel is a background candidate if luma > 0.86 AND saturation
  < 0.1. Saffron wash pixels (luma ~0.43) and black ink are retained.
- **Alpha fade:** alpha is linearly interpolated from opaque (luma 0.86) to
  fully transparent (luma 0.96). Soft edge so the saffron watercolor feathers
  naturally rather than hard-cutting.
- **Tooling:** prefers system `magick` (ImageMagick CLI) for speed; falls back
  to `sharp` (a transitive Next.js dependency) for raw RGBA buffer math. Both
  paths implement the same luma/saturation thresholds.
- Files smaller than 10KB are skipped (placeholder guard).
- Writes to `{slug}.tmp.png` then renames atomically in place.

## Backups & originals

Because transparentize rewrites in place and Gemini generation is
non-deterministic (a re-run cannot reproduce a given image), the 12 canonical
masters (`{slug}.png` + `{slug}-dark.png`) are the irreplaceable source of
truth. They are protected two ways:

- `public/illustrations/originals.zip` — a flat, git-tracked archive of all 12
  masters. The single recoverable artifact if the working copies are lost.
- `public/illustrations/_backup/` — a **gitignored**, local-only net. Holds the
  raw pre-transparentize Gemini output for the 5 slugs that were snapshotted in
  time; for `first-talk` and the `-dark` variants the raw was overwritten before
  a backup existed, so those entries are the transparentized masters instead.
  See `_backup/README.md` for per-file provenance. `_backup/regen-2026-05-25/`
  is a separate regeneration batch.

There is no original deeper than these. Treat `originals.zip` as the recovery
source and refresh it whenever a master is regenerated.

## Dark-mode variant

`{slug}-dark.png` is a separately post-processed variant where the ink
line-work has been re-tinted to warm cream so it reads on the navy night-sky
background. If the file is absent, `getIllustrationDarkUrl` falls back to the
light URL.

`SuttaIllustration` renders both images at all times and uses
`dark:hidden` / `hidden dark:block` CSS classes to toggle — this is CSS-only
and works in Server Components without hydration.

## Cache-busting

`getIllustrationUrl` reads `fs.statSync(filePath).mtimeMs` at build time and
appends `?v=<mtime_in_seconds>` to the URL. When a PNG is regenerated, the
mtime changes, the query string changes, and the browser fetches fresh.

`next.config.ts` must allowlist the pattern or Next.js `<Image>` refuses
to serve local images with query strings:

```ts
images: {
  localPatterns: [{ pathname: "/illustrations/**" }]
}
```

## Important gotchas

**`getIllustrationUrl` is Server Component only** — it calls `fs.statSync`
synchronously. Do not import it into Client Components or edge runtimes.

**Model name churn** — Google's image model endpoint names have changed with
each preview release. The script's `MODEL_CANDIDATES` array exists precisely
for this; update it when new model names are announced.

**Regenerating an illustration** — delete `public/illustrations/{slug}.png`,
run `pnpm generate-illustrations`, then `pnpm transparentize-illustrations`.
Both scripts are idempotent if files already exist.
